COMA-Det Dataset

COrruption-based Meta-dataset for AutoEval in Detection

COMA-Det is a corruption-based meta-dataset for estimating object detection mAP on unlabeled test sets under realistic camera, weather, sensor noise, and compression shifts.

Overview

COMA-Det is designed as an AutoEval evaluation meta-dataset for estimating the mAP of object detectors on unlabeled test data. Instead of relying on conventional augmentation-based meta-datasets, it builds controlled corruption subsets that reflect real deployment changes such as camera artifacts, adverse weather, sensor noise, and compression degradation.

The dataset applies ImageNet-C-style corruptions together with Cityscapes-like weather variations. It covers 10 corruption types across five scalable severity levels, allowing detector performance to be evaluated from mild visual shifts to severe degradation.

Compared with augmentation-based meta-datasets, which often look artificial and concentrate detector mAP around narrow regions near 25% and 35%, COMA-Det provides a wider and smoother mAP distribution from nearly 0% to 40%. This better matches the AutoEval goal, because an evaluation dataset with overly concentrated detection performance can make mAP prediction appear strong even when the estimator only works in that limited range.

Dataset Information

Key Application

AutoEval, object detection, unlabeled test-set evaluation, mAP estimation, corruption robustness

Primary Data Type

RGB images, 2D bounding boxes, Class labels, Corruption metadata, Severity metadata

Target Categories

  • Vehicle: sampled from COCO, BDD, Cityscapes, DETRAC, ExDark, KITTI, Self-driving, Roboflow, Udacity, and Traffic datasets
  • Pedestrian: sampled from COCO, Caltech, CrowdHuman, Cityscapes, Self-driving, ExDark, EuroCity, KITTI, and CityPersons datasets

Corruption Protocol

  • 10 corruption types: Gaussian noise, shot noise, impulse noise, defocus blur, snow, frost, fog, contrast, pixelate, and JPEG compression
  • Five severity levels for each corruption type
  • Corruptions are selected to reflect realistic camera, weather, sensor-noise, and compression shifts
  • Scalable construction supports adding new sources, corruptions, or severity schedules

Dataset Characteristics

  • Meta-dataset for evaluating mAP estimation on unlabeled object detection test sets
  • Diverse source datasets provide broader scene, camera, illumination, and domain coverage
  • Reliable labels are inherited from established detection datasets and normalized for vehicle and pedestrian evaluation
  • Corruption-based design produces a broad detector mAP range, including low-performance regions below 15%
  • Expected mAP distribution spans nearly 0% to 40%, with smoother coverage than augmentation-based alternatives

Labels

COMA-Det uses normalized detection annotations for the target classes vehicle and pedestrian. Each image keeps metadata for the source dataset, corruption type, severity level, and evaluation split. 

{
  "image_id": "bdd_000001_gaussian_noise_s3",
  "source_dataset": "BDD",
  "target_class": "vehicle",
  "corruption": "gaussian_noise",
  "severity": 3,
  "annotations": [

      "category": "vehicle",
      "bbox": [512.4, 284.1, 116.8, 72.5]
    }
  ]
